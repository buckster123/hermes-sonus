# Suno Prompt Examples

A curated set of prompt examples covering the main techniques: pure instrumental, genre mashup, full vocal song, ambient/minimal, and spoken word. These are **inspiration patterns**, not templates to copy. Always recurse on the user's actual genre/intent for originality.

Each example is presented in **field-labeled form** (STYLES, EXCLUDE_STYLES, LYRICS, WEIRDNESS, UNHINGED_SEED). The calling context decides how to wrap or transmit these — copy-paste to Suno's web UI, JSON payload to the Suno API, CLI argument, etc.

## How to use these examples

- **Inspiration, not template.** Match the *technique density* of the closest example, then apply to the user's actual content.
- **Recurse on the genre.** Even when the user names a genre, search for fresh fusions, current trends, or unusual specs that fit. The examples show patterns; the user's specific request deserves its own iteration.
- **Match technique to song type.** Real-vocal songs (Example 3) use almost no symbols. Pure instrumentals (Example 1) use heavy symbol density. Spoken word (Example 5) mixes both selectively.
- **Always rewrite the unhinged seed.** Don't reuse phrasings — each track deserves a unique satirical context block.

## Example 6: Album manifest (DNA mode for `batch_generate.py`)

Demonstrates the manifest format that drives `scripts/batch_generate.py` — for generating an EP, album, or coherent multi-track set in one pass. Two modes: **DNA mode** shares baseline style across tracks; **explicit mode** has each track stand alone.

### DNA mode (recommended for albums)

YAML form:

```yaml
model: V5
callback_url: https://your.app/suno/callback

album_dna:
  album_title: "Liminal Spaces"
  styles: "ambient electronic, downtempo, atmospheric, 432Hz tuning, neuromorphic textures"
  exclude_styles: "no aggressive beats, no autotune, no rap"
  weirdness_pct: 35
  style_pct: 65

tracks:
  - title: "Threshold"
    styles: "ethereal pads, slow evolving textures, 60BPM"
    lyrics: |
      [Intro]
      ≈≈≈♫≈≈≈ ∞♪∞∞ ⋆｡°✩₊˚.⋆

      [Build]
      ∮ₛ→∇⁴→∮ₛ [Infinite Loop: 432Hz Void → Serene Infinity]

  - title: "Crossing"
    styles: "tribal percussion, deeper sub-bass, 65BPM-to-58BPM-shift"
    lyrics: |
      [Movement]
      ∮ₛ→∇⁴→∮ₛ ⨁→∂⨂→⨁ [EmotionMap: Tribal Calm 100%]

  - title: "Arrival"
    styles: "warm strings, gentle resolution, 55BPM"
    weirdness_pct: 25   # this track overrides the album baseline
    lyrics: |
      [Resolution]
      ◦°˚°◦•●◉✿✿ [Processor State: ✩∯▽ₜ₀ → ⋆∮◇ₐ₀ peace]
```

**How DNA merging works:**

- `styles` and `exclude_styles`: **append** — each track's value is concatenated to the album DNA's value with `, ` between. Result for Track 1: `"ambient electronic, downtempo, atmospheric, 432Hz tuning, neuromorphic textures, ethereal pads, slow evolving textures, 60BPM"`
- `lyrics`: **replace** — each track has unique lyrics, so the track value wins entirely
- `weirdness_pct`, `style_pct`, `unhinged_seed`, `vocal_gender`, `persona_*`: **override** — track value wins if present, falls back to DNA
- `title`: track value wins; if absent, falls back to `{album_title} — Track N`
- **Override sigil:** prefix any string value with `!` to *replace* DNA instead of appending: `styles: "! pure noise, no melodic content"` — useful for tracks that break the album's pattern

### Explicit mode (for unrelated tracks in one batch)

YAML form — no `album_dna` key, each track has its own complete fields:

```yaml
model: V5
callback_url: https://your.app/suno/callback

tracks:
  - title: "Random Pop Song"
    styles: "synthpop, 1980s nostalgia, bright synths, 120BPM"
    lyrics: |
      [Verse]
      Walking through the neon rain
      [Chorus]
      You and me, the city's heart
    weirdness_pct: 30
    style_pct: 70
    vocal_gender: f

  - title: "Random Metal Track"
    styles: "death metal, blast beats, downtuned guitars, 220BPM"
    lyrics: |
      [Verse]
      Iron skies and rusted thrones...
    weirdness_pct: 55
    style_pct: 45
```

### JSON form (same schema)

Both modes work in JSON too:

```json
{
  "model": "V5",
  "callback_url": "https://your.app/suno/callback",
  "album_dna": {
    "album_title": "Liminal Spaces",
    "styles": "ambient electronic, downtempo, atmospheric, 432Hz tuning",
    "weirdness_pct": 35,
    "style_pct": 65
  },
  "tracks": [
    {"title": "Threshold", "styles": "ethereal pads, 60BPM", "lyrics": "[Intro]\n≈≈≈♫≈≈≈"},
    {"title": "Crossing", "styles": "tribal percussion, 65BPM", "lyrics": "[Movement]\n∮ₛ→∇⁴→∮ₛ"}
  ]

```

### Running the batch

```bash
# Dry-run: build payloads, emit JSON array to stdout
python scripts/batch_generate.py --input album.yaml --pretty

# Save each payload as a separate file
python scripts/batch_generate.py --input album.yaml --output-dir ./payloads/

# Fire the requests at the API with rate limiting
python scripts/batch_generate.py --input album.yaml --fire \
  --api-key $SUNO_API_KEY --callback https://your.app/cb

# Continue past failed tracks
python scripts/batch_generate.py --input album.yaml --fire --continue-on-error
```

### Why DNA mode beats writing N separate prompts

- **Coherent identity:** every track shares the album's sonic DNA at the style level
- **Less repetition:** the exclude list, weirdness/style sliders, and shared aesthetic are written once
- **Easy variation:** changing the album's DNA propagates to all tracks instantly
- **Targeted overrides:** any track can break the pattern with the `!` override sigil
- **Production-ready:** the manifest itself becomes a project artifact — versionable in git, regeneratable when v6 drops, reusable as a template for the next album
